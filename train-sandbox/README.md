# 🚄 おえかきトレイン（train-sandbox）

指で線路を描くと、3D の新幹線・通勤電車・SL が走り出すスマホ向け箱庭 Web アプリ。

## 遊び方

- **描くモード**: 1本指でなぞると線路になる。2本指で移動・ピンチズーム。
- 線を**交差**させると自動でポイント（分岐器）になり、列車がランダムに進路を選ぶ。
  交差角が深い（60°超）場合は分岐せず、ただの立体交差風のダイヤモンドクロッシングになる。
- 線路の周りには**木・家・ビル・駅**が自動で生える（同じ線路なら同じ街になる）。
- 行き止まりには車止めが付き、列車は自動で折り返す。
- **見るモード**: ドラッグで回転、ピンチでズーム。🎥 ボタンで列車を追いかけるカメラ。
- 🚄 / 🚃 / 🚂 ボタンで列車を追加（最大8編成）。↩️ で1本戻す、🗑️ で全消去。
- 線路と列車は localStorage に自動保存され、次回開いたとき復元される。

## 実装メモ

- Vite + TypeScript + Three.js。フレームワークなしのバニラ構成。
- 描いたストロークは Chaikin 平滑化 → 等間隔リサンプリングされ、
  全ストローク間（自己交差含む）の線分交差からジャンクションを検出して
  「エッジ＋ノード」のグラフを構築する（`src/network.ts`）。
- 列車は先頭の走行軌跡（trail）を記録し、後続車両は軌跡上を距離指定で追従する。
  これによりポイント通過・線路の描き直し・折り返しが自然に扱える（`src/trains.ts`）。
- 情景はネットワーク由来のシードで決定論的に生成（`src/scenery.ts`）。

## 開発

```bash
npm install
npm run dev      # 開発サーバ
npm run build    # 型チェック + ビルド（dist/）
```

GitHub Pages には `/my-webpage/train-sandbox/` として deploy-pages ワークフローで配信される。
