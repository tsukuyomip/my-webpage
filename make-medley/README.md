# 🎚️ Make Medley

複数の曲を読み込むと BPM と調を自動解析し、拍を揃えてクロスフェードで 1 本の
メドレーに繋ぐ Web アプリ。すべてブラウザ内（Web Audio API）で完結し、音源は
どこにも送信されません。

## できること

- **複数曲のインポート** — mp3 / wav / m4a などを複数まとめて読み込み。
- **テンポ・調の自動解析**
  - テンポ: スペクトルフラックスによるオンセット包絡線 → 自己相関で BPM と拍位相を推定。
  - 調: クロマグラム（12 音クラスのエネルギー）を Krumhansl–Schmuckler の
    キープロファイルと相関させて major / minor と主音を推定。
- **拍の可視化** — 波形上に拍グリッド（4 拍ごとに小節線を強調）を重ねて表示。
  イン / アウトのハンドルをドラッグして使用区間を選べます。
- **メドレー化** — 隣り合う曲を拍単位で揃え、等パワークロスフェードで接続。
- **BPM の合わせ方を選択**
  - **すべて揃える**: 全曲を目標 BPM にタイムストレッチ（ピッチ保持・WSOLA）。
  - **徐々に変える**: 繋ぎ目で両曲を共通のテンポカーブに沿ってワープさせ、
    BPM を滑らかに移行（DJ 的なテンポブレンド）。
- **書き出し** — プレビュー再生と 16bit WAV のダウンロード。

## ストレッチゴール（実験的）

調が異なる繋ぎ目に、短い接続メロディ（アルペジオのブリッジ）を合成して挟む
オプション。転調の唐突さをやわらげることを狙ったものです。

## 開発

```bash
npm install
npm run dev      # 開発サーバ
npm run build    # tsc 型チェック + vite ビルド
```

GitHub Pages では `/my-webpage/make-medley/` 以下に配信されます（`vite.config.ts`
の `base` を参照）。

## 実装メモ / 限界

- テンポ・調の推定はヒューリスティックで、曲によっては半分/倍テンポや平行調に
  誤検出することがあります。BPM は手入力で上書きできます。
- タイムストレッチ（WSOLA）と解析はメインスレッドで同期的に走るため、長い曲や
  多数の曲では生成中に数秒 UI が固まることがあります。
- 外部ライブラリは使わず、FFT / オンセット検出 / キー推定 / タイムストレッチは
  すべて `src/audio/` に自前実装しています。
